\begin{abstract}
A purely peer-to-peer intelligence network would allow any device to serve AI inference without relying on centralized providers. AI today depends on corporate infrastructure--users pay per query, have no choice of model, and cannot verify what served their request. We propose a system where every participant simultaneously consumes and contributes intelligence. Queries fan out to multiple independent nodes running small language models. Nodes peer-rank each other's answers, and BradleyTerry maximum likelihood estimation selects the statistically best response. A cryptographic Proof of Inference built from output logprobs prevents replay and binds proofs to node identities. A dual-track EMA reputation system tracks both answer quality and ranking honesty, with Bayesian uncertainty and adaptive validation rates. Results settle on Solana. The system requires no staking, no special hardware, and no permission to join.
\end{abstract}

\vspace{1em}

% ══════════════════════════════════════════
\section{Introduction}

AI inference has come to rely entirely on centralized providers serving as trusted intermediaries. Users send queries to corporate APIs, which run proprietary models on private hardware and charge per token. This architecture requires trust--trust that the provider runs the model it claims, trust that prompts are not stored or censored, trust that access will not be revoked or repriced.

The cost of this trust is structural. Intelligence is metered: the more you use, the more you pay. The infrastructure is concentrated: three companies control the majority of the market~\cite{ai-monopoly}. The access is unequal: most of the world's population has never used an AI tool~\cite{world-bank}. The models are closed: the best systems are proprietary, and the open-weight alternatives require hardware that individuals cannot afford.

What is needed is a system that removes the intermediary. If every device that uses AI also provides AI, the cost is distributed across all participants. The provider and the consumer become the same entity, exactly as in BitTorrent~\cite{bittorrent}, where every downloader is simultaneously an uploader.

Small language models make this possible. Models with 3--14 billion parameters now match frontier models on domain-specific tasks~\cite{phi4,qwen35,gemma3}. Ensembles of such models outperform monolithic large models on quality benchmarks~\cite{moa}. These models run on consumer laptops. What was missing is a protocol to connect them into a collective intelligence and a mechanism to ensure quality without central authority.

We propose Samhati, a peer-to-peer network where nodes run small language models, answer queries in parallel, rank each other's responses, and earn rewards proportional to quality. The protocol enforces honesty through competition, not capital.

% ══════════════════════════════════════════
\section{Swarm Inference}

A query enters the network and is classified by complexity. Easy queries dispatch to 3 nodes. Hard queries dispatch to 5 nodes and include a debate round.

Each selected node runs inference independently and returns an answer alongside a Proof of Inference (Section~6). Fan-out is parallel. A timeout drops unresponsive nodes without blocking the round. The network does not require nodes to run the same model--a round may include different architectures answering the same question. Heterogeneity strengthens the swarm: diverse models are less likely to share failure modes~\cite{diversity}.

For hard queries, a debate round follows: each node sees all other answers and rewrites its own. Multi-agent debate improves mathematical reasoning by 12.8 percentage points~\cite{debate}.

% ══════════════════════════════════════════
\section{Peer Ranking}

After answers are collected, each pair of answers is evaluated by a randomly assigned judge node. The judge returns a preference and a short reasoning chain. Judges cannot evaluate their own answers.

Judge assignment is derived from $\text{BLAKE3}(\text{timestamp} \| \text{prompt} \| N)$ and changes every round, preventing coordinated judging. For $N$ nodes, $\binom{N}{2}$ pairs are each assigned exactly one judge. This is both more secure than exhaustive evaluation (colluders cannot control assignments) and more efficient (fewer total calls).

The ranking prompt instructs judges to evaluate factual correctness and reasoning only--confidence, persuasive language, and style are explicitly excluded.

% ══════════════════════════════════════════
\section{BradleyTerry Aggregation}

Pairwise preferences are aggregated into a global ranking using the BradleyTerry model~\cite{bt}, estimated via the MM algorithm~\cite{mm}. Each node $i$ has latent strength $\beta_i$:

\begin{equation}
P(i \succ j) = \frac{e^{\beta_i}}{e^{\beta_i} + e^{\beta_j}}
\end{equation}

The algorithm iterates until convergence ($\max|\Delta\beta| < 10^{-6}$), typically in 10--25 iterations. Rankings from higher-rated judges carry more weight. The winner is $\arg\max_i \beta_i$.

% ══════════════════════════════════════════
\section{Reputation}

Each node maintains a reputation score $R \in [0, 1]$ computed from two independent tracks:

\begin{equation}
R = \alpha \cdot R_{\text{ranking}} + (1 - \alpha) \cdot R_{\text{generation}}, \quad \alpha = 0.4
\end{equation}

$R_{\text{generation}}$ measures answer quality: did this node's answer win? Updated via exponential moving average after each round:

\begin{equation}
R_{\text{gen}}^{(t+1)} = (1 - \beta) \cdot R_{\text{gen}}^{(t)} + \beta \cdot \mathbb{1}[\text{won}], \quad \beta = 0.15
\end{equation}

$R_{\text{ranking}}$ measures judgment quality: did this node's rankings agree with BradleyTerry consensus? Updated identically with accuracy as the signal. This dual-track design catches nodes that give good answers but rank dishonestly--a failure mode that single-track rating systems cannot detect.

Each node also carries an uncertainty $\sigma$, starting at 0.4 and decaying by 5\% per round to a minimum of 0.05. High uncertainty means we do not yet trust this node's rating. This is inspired by TrueSkill's Bayesian treatment of skill~\cite{trueskill}, applied here to reputation rather than game skill.

Uncertainty feeds an adaptive validation rate. New nodes with high $\sigma$ are audited in approximately 50\% of rounds. Established nodes with low $\sigma$ and high $R$ are audited in approximately 2\% of rounds:

\begin{equation}
v = v_{\max} - (v_{\max} - v_{\min}) \cdot R \cdot (1 - \sigma), \quad v_{\max} = 0.5, \; v_{\min} = 0.02
\end{equation}

This reduces verification cost by an order of magnitude without sacrificing security--an approach demonstrated by Hyperbolic's Proof of Sampling protocol~\cite{posp}.

Domain-specific $R_{\text{generation}}$ is tracked independently for Code, Math, Reasoning, and General. A node strong in Code but weak in Math will be selected for coding queries but not mathematical ones.

Nodes that fail proof verification are slashed: reputation drops to zero and the node must pass recalibration before re-entering the swarm. Inactive nodes decay at 2\% per round of absence.

% ══════════════════════════════════════════
\section{Proof of Inference}

Each node produces a cryptographic proof from its output logprobs. For each generated token, the node records the token ID and top-8 probability values. Tokens are grouped into 32-token chunks and each chunk is hashed with BLAKE3 from a deterministic serialization. The signable message is:

\begin{center}
\texttt{model\_hash $\|$ token\_count $\|$ chunk\_hashes $\|$ timestamp $\|$ node\_pubkey}
\end{center}

signed with Ed25519. The verifier checks seven properties: model hash matches, token count matches, chunk count equals $\lceil\text{tokens}/32\rceil$, indices are sequential, timestamp is within 5 minutes, node is registered, and signature is valid.

This proof does not verify internal computation--it does not hash layer activations or confirm which model ran. It verifies that \textit{a} model produced specific output distributions, signed by a specific node, recently. Replay is prevented by the timestamp. Proof theft is prevented by the node binding. Response tampering is prevented by the signature covering the logprob hashes.

The proof mechanism handles authenticity. The ranking mechanism (Section~3) handles quality. Together they ensure that a node must run real inference and produce a good answer to earn rewards.

% ══════════════════════════════════════════
\section{Why Not Proof of Stake}

Existing decentralized inference networks require nodes to stake tokens before participating~\cite{fortytwo}. A node that loses a round forfeits its stake.

This creates a capital barrier. The operator must acquire tokens before contributing compute. In practice, this excludes participants who have hardware but not cryptocurrency--the inverse of the problem decentralization should solve.

Staking optimizes for capital. A well-funded node with a poor model can afford to participate indefinitely. A skilled operator with a good model but no tokens cannot participate at all.

Samhati replaces economic proof with computational proof. Producing a valid Proof of Inference requires running real inference. Producing a winning answer requires a model good enough to beat peers in ranking. The barrier is a laptop, not a wallet. The cost of attack is electricity, not money--and electricity, unlike tokens, cannot be borrowed.

% ══════════════════════════════════════════
\section{Network}

Nodes discover each other through two mechanisms: an on-chain registry on Solana~\cite{solana} and a gossip protocol over iroh~\cite{iroh} QUIC connections.

A single Ed25519 keypair derives the node's Solana address, iroh peer identity, and proof signing key. On startup, the node registers on Solana (creating a persistent identity), reads all registered nodes (a free RPC call), connects via iroh with relay-assisted NAT traversal, and joins a gossip topic. Every 10 seconds it broadcasts its model name and inference endpoint. New peers are verified against Solana before acceptance.

% ══════════════════════════════════════════
\section{Incentive}

Nodes earn SMTI tokens for each round. The winner receives 60\% of the base emission. The remaining 40\% is distributed proportionally by reputation score. Domain specialists earn a $1.5\times$ multiplier on matching queries.

Domain demand counters are tracked on-chain per round. If Code queries dominate, operators see the signal and run Code specialists. The network self-balances through transparent demand, not central allocation.

There is no fee to participate. No stake to post. No token to acquire. The economic defense against Sybil attacks is the compute cost of producing valid proofs and competitive answers.

% ══════════════════════════════════════════
\section{Settlement}

Each round settles on Solana via a single transaction containing participant keys, proof hashes, reputation deltas, winner identity, domain tag, and emission amount. Three program-derived accounts store network state:

\begin{center}
\begin{tabular}{@{}lll@{}}
\toprule
\texttt{ProtocolConfig} & \texttt{[b"config"]} & emission rate, demand counters \\
\texttt{NodeAccount} & \texttt{[b"node", pubkey]} & reputation, model, rounds, rewards \\
\texttt{RoundAccount} & \texttt{[b"round", id]} & participants, proofs, deltas \\
\bottomrule
\end{tabular}
\end{center}

% ══════════════════════════════════════════
\section{Security}

\textbf{Sybil attacks.} Each fake node must produce valid logprobs, requiring real inference. Low-quality answers lose reputation and stop being selected. The cost of maintaining fake nodes exceeds the reward from gaming the ranking.

\textbf{Collusion.} Judge assignments change every round and cannot be predicted. The probability that all judges for a colluder's pairs are also colluders drops exponentially with honest participation.

\textbf{Replay.} Proofs expire in 5 minutes and are bound to a unique public key. A proof from node $A$ cannot be submitted by node $B$.

\textbf{Freeloading.} A node running a weak model will consistently lose in ranking. Its $R_{\text{generation}}$ decays via EMA, reducing both selection probability and reward share. Quality self-selects without model verification.

\textbf{Self-promotion.} Judges evaluate factual correctness only. Persuasive language is excluded by prompt design. Nodes cannot judge their own answers.

% ══════════════════════════════════════════
\section{Conclusion}

We have proposed a peer-to-peer intelligence network where every participant simultaneously uses and provides AI inference. Queries fan out to multiple nodes running small language models. Peer ranking selects the best answer via BradleyTerry. Dual-track EMA reputation tracks both answer quality and ranking honesty. Cryptographic proofs prevent replay and tampering. Random judge assignment prevents collusion. Results settle on Solana.

The system requires no staking, no special hardware, and no permission to join. Quality is enforced by competition, not capital. The barrier to entry is a laptop, not a wallet.
